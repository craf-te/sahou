# Sahou × TouchDesigner — custom operators

> **Status: experimental.** The Sahou Out CHOP is currently **macOS / arm64 only** and
> validates channels against the schema (with optional Zenoh sending). Building it
> requires the TouchDesigner C++ SDK vendored into `td/vendor/` (Derivative Shared Use
> License — not redistributed here; copy it from your local TouchDesigner install).

TouchDesigner as a first-class Sahou node: send/receive over the typed contract from custom
C++ operators, **without going through TD's Python** (thin C++ glue + the Rust core, which is
statically linked).

## Status

- **Sahou Out CHOP** (macOS arm64). Reads the input CHOP's channels, projects them to a JSON
  payload, and runs the **send boundary** through the Rust core (`sahou_prepare_publish`). On a
  contract violation the node goes red with the structured diagnostic ("say NO in the right place").
- **Test Send** — a pulse that publishes one IR-valid sample of the selected connection over
  **Zenoh** (via the bundled `libsahou_transport`), for a quick connectivity check with `sahou tap`.
  This is the first real transmit; **continuous per-frame send + QoS mapping + on-change dedup**
  (the `Active` / `Send Every Cook` toggles) are the next stage.
- Not yet: continuous send, Sahou In CHOP, DAT operators, Windows build, universal (Intel) binary,
  distribution signing (multicast entitlement / Developer ID).

## Layout

```
td/
├── src/                 # our code (committed)
│   ├── SahouOutCHOP.{h,cpp}   the op — thin TD glue
│   ├── payload.{h,cpp}        pure: channels -> JSON payload (no TD SDK)
│   └── envelope.{h,cpp}       pure: read the core's OK/NO envelope for status
├── test/                # TD-independent tests (committed)
│   ├── payload_test.cpp / envelope_test.cpp   pure unit tests
│   ├── ffi_smoke.cpp                          payload -> core -> OK/NO (links the C ABI)
│   └── run.sh                                 builds & runs all three
├── macos/               # xcodeproj + Info.plist (committed)
├── transport/           # sahou-transport: zenoh send glue (Rust cdylib, committed) — bundled into
│                         # the .plugin's Contents/Frameworks and ad-hoc signed by build-td-macos
├── examples/            # schema.sahou.yaml (committed); gen/descriptor.json (generated, ignored)
├── vendor/              # TD C++ SDK — Derivative-licensed, NOT committed (see below)
└── build/               # xcodebuild output — ignored
```

## Prerequisite — vendor the TD C++ SDK (once)

The TD C++ SDK (`CHOP_CPlusPlusBase.h`, `CPlusPlus_Common.h`) and the sample project are owned by
Derivative under the **Shared Use License**, so they are **not committed** (`td/.gitignore`). Copy
them from your local TD install into `td/vendor/`:

```sh
cp -R "/Applications/TouchDesigner.app/Contents/Resources/tfs/Samples/CPlusPlus/CHOP/." td/vendor/
```

(If your TD ships without the samples, get them from
[TouchDesigner/CustomOperatorSamples](https://github.com/TouchDesigner/CustomOperatorSamples).)
The xcodeproj expects the headers and `Info.plist` template location under `td/vendor/` and the
op's own `Info.plist` under `td/macos/`.

## Build (macOS, arm64)

```sh
just build-ffi        # target/release/libsahou_core.a + core/sahou.h (once / after core changes)
just build-td-macos   # -> td/build/Release/SahouOut.plugin
```

`build-td-macos` runs `xcodebuild` on `td/macos/SahouOut.xcodeproj`. The Rust core is **statically
linked**, so the `.plugin` is self-contained — unlike the spike there is no companion `sahou_td`
dylib to co-locate. The build is pinned to `arm64` (TD 2025 on Apple Silicon); a universal binary
for Intel TD would need a `lipo`'d `libsahou_core.a` (follow-up).

## Test

```sh
just test-td   # pure unit tests (payload/envelope) + FFI smoke (payload -> core -> OK/NO)
```

## Load in TouchDesigner

TD scans, in order: the per-`.toe` `Plugins/` folder next to your project; the user-global
`~/Library/Application Support/Derivative/TouchDesigner099/Plugins/`; and `$TOUCHDESIGNER_PLUGIN_PATH`.

During development, **symlink** the built bundle instead of copying — then a rebuild only needs a
TD restart (no re-copy):

```sh
PLUGINS="$HOME/Library/Application Support/Derivative/TouchDesigner099/Plugins"
mkdir -p "$PLUGINS"
ln -s "$PWD/td/build/Release/SahouOut.plugin" "$PLUGINS/SahouOut.plugin"
```

Then restart TD (plugins are scanned at startup; a running TD locks the loaded bundle) and add a
**Sahou Out** CHOP. Notes: the symlink target lives under `td/build/` (git-ignored), so a
`cargo clean` / removed build breaks the link — just rebuild. The local ad-hoc signature means no
Gatekeeper quarantine.

## Use

Parameters:

- **IR File** — path to a Sahou `descriptor.json` (generated by `sahou gen`).
- **Node** — sending node. **Dropdown** populated from the IR (nodes that are the `from` of a
  pub_sub connection). Disabled until an IR loads.
- **Connection** — connection to publish on. **Dropdown** of the selected Node's pub_sub
  connections. Disabled until a Node is picked. (No family-fit filter yet: a non-numeric
  connection can be picked and will simply report a boundary NO on send — follow-up.)
- **Cook Every Frame** — like the OSC Out CHOP's toggle. **On (default)** because TD is pull-based:
  a send/sink node with nothing pulling its output would otherwise miss input changes. Turn Off only
  when something downstream pulls this node every frame. (Avoiding redundant traffic when the input
  is idle is the job of a `Send Every Cook`/on-change toggle at the send stage, not of not cooking —
  that toggle plus `Active` will be added here when Zenoh send is wired.)
- **Reload** — re-read the IR File (after regenerating it).
- **Test Send** — publish one IR-valid sample of the selected connection over Zenoh (works with no
  input). Verify receipt with `sahou tap`:
  ```sh
  sahou tap td/examples/gen/descriptor.json --node viz --count 1   # subscribe to `motion` as `viz`
  ```
  then press **Test Send** in TD; the tap prints `[motion] #0 OK sahou/motion {...}`. The node's
  Info DAT `test` row shows the sent payload and transport status (`opened` / `sent` / `error`).

All three values are saved in the `.toe`/`.tox` and restored on open (standard TD custom-parameter
behavior); on load the IR is re-read and the menus are rebuilt so the saved Node/Connection
re-select. Use an absolute IR File path if you move the `.toe` around.

Wire an input CHOP; each channel becomes a field of the message (channel name → field name,
latest sample → value). The node's status shows up as:

- **red error** with `code at path: message` on a contract violation (missing required field,
  wrong type, unknown connection, …),
- a **yellow warning that tells you what to wire** before an input is connected, e.g.
  `expected channels: x(float), y(float), speed(float)` (optional fields marked `?`),
- an **Info CHOP** (`valid` = 1/0/-1, `seq`, `execute_count`),
- an **Info DAT**: first the status (`status`, `node`, `connection`, `keyexpr`, `schema_hash`,
  `detail`), then the **expected schema of the selected connection** as a table
  (`field | type | required | detail`) — so you can see exactly what to send. Attach an `Info DAT`
  operator to the node to view it.

### Demo contract

`td/examples/schema.sahou.yaml` defines node `td` / connection `motion` with three float fields
`x`, `y`, `speed`. Generate its descriptor and point the op at it:

```sh
just gen-td-demo   # -> td/examples/gen/descriptor.json
```

Set **Node** = `td`, **Connection** = `motion`, and feed a CHOP with channels named `x`, `y`,
`speed`. Rename one (e.g. drop `speed`) to watch the node go red with `required at
connections.motion.payload.speed: missing required field 'speed'`.
